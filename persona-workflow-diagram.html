5

Approve the proposed fix

The PM card doesn't just describe the job — it proposes running it. Clicking Approve executes that one pre-vetted action immediately, despite the session being read-only.

POST /api/remediation/:id/approve → executed
6

Ask a cross-module follow-up

"Any assets missing agent coverage?" routes to Cloud Agent inventory and flags two blind spots — visible only because CA is in scope.

POST /api/chat → module CA
7

Try a write command anyway

Sending an arbitrary patch-apply command through the generic passthrough is rejected outright — DENY_WRITE blocks it unconditionally. Only the one named, evidence-backed proposal from step 5 was ever allowed to run.

POST /api/qualys-cli → 403 blocked
8

Open the audit log

Every call this session made — ranking, evidence, chat, the approve → execute pair, the blocked write — is listed with timestamp, tool, module, and result.

GET /api/audit
Leaves with a cross-module remediation picture, one fix actually shipped under approval — and a defensible audit trail of how all of it was built.

Expand the same asset

Only VM and WAS cards render. A banner takes the place of the rest: "3 modules hidden under the manager allowlist (PC, CA, PM)." The Approve/Reject action from the Analyst lane never renders here — it lives on the PM card, and PM is one of the hidden three.

GET /api/assets/checkout-web/evidence · 2/5 visible

Ask why it's risky

5

GET /api/audit · CSAM / VM / WAS only
